**Goal:** Add the first GitHub Actions quality gate for `@jaemin/eslint-config` by running the existing `pnpm check` script on pull requests and pushes to `main`.

**Architecture:** Add one workflow file that delegates verification to the repository's existing package script instead of duplicating check commands in CI. The workflow uses Node 24, enables Corepack, installs dependencies with the lockfile frozen, and treats any command failure as a blocking CI failure.

**Tech Stack:** GitHub Actions, Node.js 24, Corepack, pnpm 10, existing `pnpm check` script.

## Task 1: Add Minimal GitHub Actions Check Workflow

**Files:**

- [ ] **Step 1: Create the workflow directory**

Run:

```bash

```

Expected: `.github/workflows` exists.

- [ ] **Step 2: Add the workflow file**

Create `/Users/jaemin/programming/projects/active/eslint-config/.github/workflows/check.yml`:

```yaml
name: Check

on:
  pull_request:
  push:
    branches:
      - main

permissions:
  contents: read

jobs:
  check:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v7.0.0

      - name: Setup Node
        uses: actions/setup-node@v6.4.0
        with:
          node-version: 24

      - name: Enable Corepack
        run: corepack enable

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Run checks
        run: pnpm check
```

Expected: The workflow has exactly one job, runs on pull requests and pushes to `main`, and delegates verification to `pnpm check`.
